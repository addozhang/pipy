## Description

Appends a **handleMessageEnd** filter to the current pipeline layout.

A **handleMessageEnd** filter calls a user function every time it sees a [MessageEnd](/reference/api/MessageEnd) event in the input stream. Its input and output can be any kinds of events.

<FilterDiagram
  name="handleMessageEnd"
  input="Event"
  output="Event"
/>

The user function that gets called will have a [MessageEnd](/reference/api/MessageEnd) object as parameter.

This filter passes every input event down to its output. It doesn't change how many or what events there are. However, since events are all just objects being passed around by reference, if you alter the event object itself, the change will be visible on the output, even when the references to the events are still identical between the input and the output.

## Syntax

``` js
pipy()
  .pipeline('example')
    .handleMessageEnd(
      event => onMessageEnd(event)
    )
```
